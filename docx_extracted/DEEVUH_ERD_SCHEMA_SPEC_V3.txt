This document defines the MVP database schema, entity relationships, and core data structures for the Deevuh fashion e-commerce platform.

2. Core Database Philosophy
Lean relational schema optimized for MVP launch
Supports future scaling without unnecessary complexity
Normalized structure for maintainability
Transactional consistency for orders and payments
Simple operational workflows for admin management

4. Relationship Definitions
One user can have multiple carts
One cart can contain multiple cart items
One product can have multiple variants
One product can have multiple images
One order can contain multiple order items
One user can place multiple orders
Cart items and order items reference product variants
Coupons can be applied during checkout
5. MVP Optimization Decisions
No microservice-specific schema separation
No event-store architecture
No advanced analytics tables initially
No Redis-dependent schema logic
Minimal joins for admin dashboard efficiency
6. Future Expansion Readiness
Wishlist tables
Reviews and ratings
Inventory logs
Marketing automation
Loyalty systems
Advanced analytics events
Multi-address support
Shipment tracking
7. Final Technical Direction
The data layer is intentionally designed to remain simple, maintainable, and operationally efficient for the early-stage launch of the Deevuh platform. The schema supports the current business requirements while preserving a clean upgrade path for future scaling.
